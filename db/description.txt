
1. Проект сделан для БД PostgresSQL

2. Настройки проекта предполагают
подключение проекта к БД c названием
task_back

3. В силу некоторых причин, например,
- проект маленький
- проект больше учебный
- проект может потребовать экспериментов
я не стал включать в проект работу c Liquibase.
В больших проектах конечно лучше использовать
специализированные системы

4. Все действия с таблицами надо выполнять руками.
То есть если вы  создали БД и подключились
к ней, то вы можете запустить нужные вам
скрипты для создания и заполнения БД данными
и далее запускать проект.

5. Помимо основных скриптов в файлы sql добавлены
дополнительные скрипты (как правило для очистки даных)
Не запускайте все сразу, запускайте только то,
что нужно сейчас
